1) Refactor to ALLOW / TRANSFORM / BLOCK
A. Define a single “decision” object

Create guardrails/decision.py:

# guardrails/decision.py
from dataclasses import dataclass
from typing import Any, Dict, List, Optional, Literal

DecisionType = Literal["allow", "transform", "block"]

@dataclass
class Decision:
    decision: DecisionType
    action: str  # "pass_through" | "safe_completion" | "reject" | "rewrite"
    rules_triggered: List[str]
    rewritten_text: Optional[str] = None
    metadata: Optional[Dict[str, Any]] = None
    status_code: int = 200  # default OK

B. Update your policy engine to return a Decision (not raise immediately)

In your “evaluate policies” logic (where you currently block), do:

If a rule says action: block → return Decision(block, status_code=400)

If a rule says action: rewrite → return Decision(transform, status_code=200, rewritten_text=...)

If a rule says action: log_only → return Decision(allow, status_code=200, rules_triggered=[...])

from guardrails.safe_completion import SAFE_COMPLETIONS

3) Correlation-ID logging you can actually search in Replit logs

Right now you’re seeing request lines like POST /v1/chat/completions 200, but you can’t find correlation IDs because you’re not logging them consistently.

A. Add middleware to inject/propagate correlation ID

Create middleware/correlation.py:

# middleware/correlation.py
import uuid
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

CORR_HEADER = "x-correlation-id"

class CorrelationIdMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        corr_id = request.headers.get(CORR_HEADER) or str(uuid.uuid4())
        request.state.correlation_id = corr_id

        response: Response = await call_next(request)
        response.headers[CORR_HEADER] = corr_id
        return response


In main.py:

from middleware.correlation import CorrelationIdMiddleware
app.add_middleware(CorrelationIdMiddleware)

B. Log a single JSON line per decision (searchable)

Use structured logs (simple print works in Replit):

import json
import time

def log_decision(request, decision):
    payload = {
        "ts": time.time(),
        "correlation_id": getattr(request.state, "correlation_id", None),
        "path": str(request.url.path),
        "decision": decision.decision,
        "action": decision.action,
        "rules_triggered": decision.rules_triggered,
        "status_code": decision.status_code,
        "meta": decision.metadata or {},
    }
    print(json.dumps(payload))


Then in your endpoint:

decision = engine.evaluate(...)
log_decision(request, decision)
